2. Pipeline Architecture (v4 Target)

Python (deterministic)
Opus (synthesis, 1M context)
Sonnet (isolated research)
Human checkpoint
Phase 0
Intake

Control Panel + Research Brief

22 parameters: Topic, Decision, Audience, Risk Tier, Freshness, Narrative Arc, Original Thesis target

→ control-panel.json, research-brief.json

Phase 1
Research

Multi-Source Investigation (5 isolated Sonnet agents)

Each sub-question → independent Sonnet with web_search. Source Log + Claim Ledger + Contradiction Register + Gap Map. Source diversity check: ≤60% from any category.

→ source-log.md, claim-ledger.md, contradictions.json, gap-map.md, all-claims.json

Law 2: Sonnet speaks only to Python. Never to Opus. Never to other Sonnet.

Phase 1.5
RAG Enrich

Hierarchical Knowledge Retrieval

Query RAG with each sub-question + each claim. CORE (2x) hits override EPHEMERAL (0.5x). Merge with Sonnet findings. Flag conflicts between RAG knowledge and fresh research.

→ enriched-claims.json (claims + RAG evidence + tier provenance)

Law 3: Python everywhere deterministic.

⚡ Human Checkpoint 1 (30 seconds): Review enriched claims + gap map. Approve or redirect.

Phase 2
Thesis

Thesis Development (1 Opus call)

Original thesis ("Would an expert disagree?"), proposed framework (named), constructed scenario (3+ empirical steps), narrative arc (A→B transformation), section titles as arguments, 3× "Nobody else is saying"

→ thesis.json (thesis, framework, scenario, arc, titles, nobody_bullets)

Law 1: Opus receives only. Invents no facts. Connects what it gets.

Phase 3
Outline

JSON Outline (1 Opus call)

Section structure driven by thesis.json narrative arc. Each section: title (argument!), assigned claims, assigned sources, min words, KPI candidates, invalidation prompt. Claims assigned to Appendix only (not prose sections).

→ outline.json

Phase 4
Sections

Per-Section Writing (7-12 Opus calls, sequential)

Each section gets: outline contract + assigned claims + RAG chunks for that section + all previous sections (consistency). Persona: "You are Florian Ziesche, founder of Ainary Ventures. Direct, no filler."

Per-section validation (Python): word count, assigned sources present, banned words, "For the decision maker" callout, section confidence %, E/I/J/A ratio. Retry once if fails.

→ sections/*.md (each section independently)

Phase 5
Validate

13 Deterministic Checks

E/I/J/A count (E>50%, J<20%) · 3-Signal Confidence per claim · Beipackzettel correction · Source orphan check · Cross-reference check · Grounding check · Consistency check (numbers across sections) · Knowledge provenance chain

→ validation-report.json

Phase 6
Merge

Assemble + Grade + Beipackzettel + HTML

Merge sections into single document. Compute final grade (A+++/A++/A+/B). Generate Beipackzettel (confidence, risk level, blind spots, not checked). Insert into AR-001 HTML template with KPI grids, callout boxes, exhibits, cover, TOC, back cover. Knowledge hierarchy visible in Source Log.

→ final-report.md, final-report.html, final-report.pdf, beipackzettel.json, trust-score.json

⚡ Human Checkpoint 2 (2 minutes): Read Beipackzettel + Executive Summary + Section Titles. Approve, edit titles, or request changes.

5. Data Flow: Knowledge → Report → Knowledge

INPUT (Knowledge Hierarchy)
┌─ CORE (2x): CRTs + Corrections + Reference Library
├─ KNOWLEDGE (1.5x): Vault Papers + Developments + Patterns
├─ OPERATIONAL (1x): Projects + People + AR-INDEX
└─ EPHEMERAL (0.5x): Daily notes + Session logs

               ↓ RAG Query (per section)

PIPELINE (9 Phases)
  Sonnet Research → Python Enrich → Opus Thesis → Opus Outline
  → Opus Sections → Python Validate → Python Merge + HTML

               ↓ Report Output

OUTPUT
┌─ Report (HTML + PDF + MD) with Beipackzettel
├─ beipackzettel.json (machine-readable)
├─ trust-score.json (cumulative agent trust)
└─ validation-report.json (13 checks)

               ↓ Feedback Loop

WRITEBACK (Knowledge grows)
┌─ New claims → Claim Registry (OPERATIONAL)
├─ Verified findings → CRTs (CORE) ← human approval
├─ Corrections → corrections.json (CORE)
├─ Trust Score update → trust-score.json
└─ Predictions → Prediction Scorecard (tracked monthly)
