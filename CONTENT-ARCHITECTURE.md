# PART 5: CONTENT STANDARDS

## 5.1 Voice Guidelines

**Position:** The Technical Operator Who Shows the Work

**Voice Formula:**
- 65% Casual, 35% Corporate
- 70% Technical, 30% Accessible
- 55% Bold, 45% Humble

**The Voice Test:** Before publishing, read the copy aloud. Does it sound like a technical founder talking to a peer at a conference bar ‚Äî not a keynote stage, not a marketing email? If yes, it's right.

### Writing Principles

**1. Lead with specifics, not claims.**
```
‚úÖ "<0.2% hallucination rate across 10,000+ documents"
‚ùå "Industry-leading accuracy"
```

**2. Active voice. First person. Short sentences.**
```
‚úÖ "We built a multi-agent system. It processes contracts in 3 minutes."
‚ùå "A multi-agent system was developed to enable efficient processing."
```

**3. Admit what you don't know.**
```
‚úÖ "Our legal AI works for contract analysis. We haven't tested it on litigation discovery yet."
‚ùå "Our AI works for all legal use cases."
```

**4. Use numbers, not adjectives.**
```
‚úÖ "70% reduction in planning time. ‚Ç¨180K annual savings."
‚ùå "Significant cost reduction and improved efficiency."
```

**5. One idea per paragraph. One CTA per section.**

**6. Avoid the corporate immune system.**
```
‚úÖ "We build AI. Not pitch decks about AI."
‚ùå "We leverage cutting-edge artificial intelligence solutions."
```
