**Tweet 3:**
The biggest mental shift:

Stop thinking "AI tool I use sometimes."

Start thinking "team member with a job description."

When I wrote actual role descriptions for each agent, the output quality jumped immediately.

**Tweet 4:**
How I onboard a new AI agent:

1. Define the role (what, not how)
2. Give it context (who am I, what's the business, what matters)
3. Set boundaries (what it can do alone vs. what needs my approval)
4. Start with zero autonomy
5. Expand permissions as trust builds

Same process I used for human hires. Works just as well.

**Tweet 5:**
The performance review is real.

Every week I check:
- Did the agent complete its tasks?
- Quality of output (1-5 scale)
- False positives / errors
- Time saved vs. doing it myself

Bad performance = system update.
Good performance = more autonomy.

**Tweet 6:**
Mistake I made early: giving agents too much freedom.

My outreach agent sent 40 emails with a tone that didn't sound like me. No damage done, but I learned: new agents get training wheels.

Now: first 50 outputs get manual review. No exceptions.
