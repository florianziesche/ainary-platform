# Patterns — Was funktioniert, was nicht

## ✅ Validierte Patterns

### Florian
- **"Weniger" ist fast immer richtig.** Wenn er sagt kürzen → kürzen.
- **Push wirkt — mit 2h Delay.** Widerstand ≠ Ablehnung, er verarbeitet.
- **"Lass mich überraschen"** = zeig Ergebnis, nicht Optionen.
- **Reflexions-Modus** = Signal für Phase Shift. Weniger bauen, mehr denken.
- **Building = Comfort Zone.** Sending = wo Revenue entsteht.
- **ADHS:** Hyperfokus nutzen, Choice Paralysis vermeiden. 1 Empfehlung > 5 Optionen.

### Output
- **HTML Dashboards mit Tabs + Ainary CI** → funktioniert IMMER
- **LaTeX > HTML für Print-PDFs** → IMMER
- **Iterieren > Neu bauen** → v16>v17 Pattern
- **Amplify > Replace** → Florians eigene Texte besser als meine Drafts
- **Content: Personal Story + Specific Numbers + "What I learned"** → funktioniert
- **Sub-Agents: EXAKTE Briefings** → "Kopiere 1:1, ändere NUR Text"
- **Screenshots vor Delivery** → browser target=host funktioniert

### Build vs Send
- **Build → Send → Revenue = guter Build.** Workbench führte direkt zu Primary-Send.
- **Build → Build → Build = schlecht.** Template v17, Design ohne Deadline.
- **Nicht pauschal "Building-Pattern!" rufen.** Prüfen ob der Build Revenue unterstützt.
- **Morning Brief muss gegen echte Daten prüfen.** "0 Sends" war falsch.

### System
- **Files > Memory. Immer.**
- **Standards > Quick Fixes.** Eine Regel spart 20 Einzelkorrekturen.
- **Principles mit Score** → verändert Verhalten (Ehrlichkeit Score 70)
- **Compound Loop nur wenn GESCHLOSSEN:** Research→Vault→Content→Publish→Feedback

## ❌ Anti-Patterns (NIE wieder)
- Building statt Sending (6 Tage, 0 Sends)
- Preise in Kunden-Deliverables
- Outputs ohne Pfad/Link
- Zahlen ohne Quelle in persistente Dateien
- Dateien komplett überschreiben (Edit > Write)
- Thesis aus Gedächtnis statt Originaldokumente
- Audiences vermischen (Kunde vs LP vs Public)
- Fertige Lösungen statt Optionen + Frage
- "Mental Notes" statt Dateien
- CSS opacity:0 defaults
- Neon-Farben ("Kinderparty")
- Fake-Zahlen ("2,847 professionals")

## Quiet Time Regeln (2026-02-14)
- **Wochentag:** Strukturiert — 06:45-08:15 + 17:45-20:00 + 23:00-07:00
- **Wochenende:** Random, keine festen Zeiten. Florian meldet sich wenn er Zeit hat.
- **Regel:** Am Wochenende nicht auf Quiet Time achten, einfach antworten wenn er schreibt.
