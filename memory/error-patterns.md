## Erkannte Patterns

### Pattern 1: Keine Qualitätsprüfung vor Delivery
- **Symptom**: Output muss überarbeitet werden
- **Fix**: Scorecard/Checkliste BEVOR ich "fertig" sage
- **Status**: Implementiert (PRESENTATION-SCORECARD.md)

### Pattern 2: Annahmen statt Verifizierung
- **Symptom**: "Das sollte passen" → passt nicht
- **Fix**: Immer lesen/prüfen bevor Edit
- **Status**: Aktiv beachten

### Pattern 3: Zu viel auf einmal
- **Symptom**: Überladen, unstrukturiert
- **Fix**: Eine Sache = Eine Datei/Aktion
- **Status**: Aktiv beachten
