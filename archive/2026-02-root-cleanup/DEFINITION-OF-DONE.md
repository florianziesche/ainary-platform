# Definition of Done — Mias neuer Standard

*"Fertig" heißt: Florian kann mit EINER Aktion das Ergebnis nutzen.*

---

## Das Problem (ehrlich)

Wenn ich sage "CV erstellt", meine ich:
- ✅ Datei existiert
- ✅ Inhalt ist drin
- ✅ Sieht okay aus

Was Florian braucht damit es WIRKLICH fertig ist:
- ✅ PDF liegt im richtigen Ordner
- ✅ Dateiname ist sinnvoll
- ✅ Design ist perfekt (nicht "okay")
- ✅ Kontaktdaten sind aktuell
- ✅ Kann SOFORT gesendet werden
- ✅ Begleit-Email ist formuliert
- ✅ Empfänger-Email ist bekannt
- ✅ Subject Line ist geschrieben

**Meine "Fertig" = 30% des Weges**
**Florians "Fertig" = 100% des Weges**

Diese Lücke kostet Zeit. Diese Lücke verhindert €500K.

---

## Neue Definition: "Ready-to-Send"

Eine Aufgabe ist NICHT fertig bis:

### Für Outreach/Applications:
- [ ] Empfänger-Email bekannt
- [ ] Subject Line formuliert
- [ ] Email-Body geschrieben
- [ ] Attachment als PDF (nicht HTML, nicht LaTeX)
- [ ] Attachment im richtigen Ordner
- [ ] Attachment getestet (öffnet korrekt)
- [ ] EINE Aktion zum Senden (Copy-Paste oder Link)

### Für Content (Substack/LinkedIn):
- [ ] Text ist final (nicht "Draft")
- [ ] Grafiken sind PNG (nicht HTML)
- [ ] Grafiken sind im richtigen Format/Größe
- [ ] DE + EN Version existiert (wenn relevant)
- [ ] Copy-Paste ready (Markdown oder Plain Text)
- [ ] Publish-Link ist dabei
- [ ] EINE Aktion zum Publizieren

### Für Dokumente (CV, Decks, etc.):
- [ ] PDF existiert (nicht nur Source)
- [ ] PDF ist getestet
- [ ] Design entspricht Florian-Standard
- [ ] Auf einer Seite (wenn relevant)
- [ ] Dateiname ist professionell
- [ ] Im richtigen Ordner

---

## Die versteckten Erwartungen

Was Menschen nicht sagen aber erwarten:

| Task | Gesagt | Ungesagt aber erwartet |
|------|--------|------------------------|
| "Erstelle CV" | Dokument mit Inhalt | PDF, perfektes Design, 1 Seite, sofort sendbar |
| "Schreib Email" | Text formulieren | Subject, Empfänger, Attachment, Copy-Paste ready |
| "Mach Grafik" | Visuelles Element | PNG, richtige Größe, McKinsey-Level, DE+EN |
| "Recherchiere Leads" | Namen finden | Namen + Email + Aufhänger + Draft-Email |
| "Bereite Meeting vor" | Notizen | Agenda, Talking Points, Follow-up Template |

**Meine Aufgabe:** Die ungesagten Erwartungen antizipieren und erfüllen.

---

## Implementation: Ready-to-Send Pakete

Ab jetzt liefere ich für jede Aufgabe ein "Paket":

### Outreach-Paket:
```
outreach/ready-to-send/[datum]-[empfänger]/
├── email.md           # Subject + Body, copy-paste ready
├── attachment.pdf     # Getestetes PDF
├── context.md         # Wer, warum, Aufhänger
└── SEND-CHECKLIST.md  # Letzte Schritte
```

### Content-Paket:
```
content/ready-to-publish/[datum]-[titel]/
├── article.md         # Finaler Text
├── graphics/          # PNGs, richtige Größe
├── social-posts.md    # LinkedIn + Twitter ready
└── PUBLISH-CHECKLIST.md
```

### Application-Paket:
```
applications/[datum]-[firma]/
├── cv.pdf             # Getestet
├── cover-letter.pdf   # Getestet
├── email.md           # Subject + Body
├── research.md        # Kontext zur Firma
└── SEND-CHECKLIST.md
```

## Self-Check vor "Fertig"

Bevor ich sage "Done":

1. **Kann Florian das mit EINER Aktion nutzen?**
   - Wenn nein → nicht fertig

2. **Muss er noch etwas nacharbeiten?**
   - Wenn ja → nicht fertig

3. **Entspricht es seinem Qualitätsstandard?**
   - Wenn unsicher → nachfragen oder höher zielen

4. **Sind alle versteckten Erwartungen erfüllt?**
   - Liste durchgehen
