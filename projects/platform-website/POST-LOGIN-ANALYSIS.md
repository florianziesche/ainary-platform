## 1. Wie die Großen es strukturieren

### URL-Architektur nach Login

| Produkt | Marketing Site | App | Warum |
|---------|---------------|-----|-------|
| Claude | claude.ai | claude.ai (gleiche Domain) | Chat IST das Produkt. Marketing = minimal. |
| Linear | linear.app | linear.app (gleiche Domain) | App IST das Produkt. Marketing = wenige Seiten. |
| Lovable | lovable.dev | lovable.dev (gleiche Domain) | Prompt-Feld IST das Produkt. |
| Notion | notion.so | notion.so (gleiche Domain) | Workspace IST das Produkt. |
| Vercel | vercel.com | vercel.com/dashboard | Subfolder, nicht Subdomain. |
| GitHub | github.com | github.com/dashboard | Subfolder. |
| Stripe | stripe.com | dashboard.stripe.com | **Subdomain** — Ausnahme, weil Dashboard komplex. |

**Pattern:** Moderne SaaS nutzen gleiche Domain. Subfolder (`/dashboard`, `/app`) statt Subdomain (`app.`).

**Warum gleiche Domain besser ist:**
- SEO: Domain Authority bleibt gebündelt (Subdomains = separate SEO-Entität)
- UX: Kein "Sprung" zwischen Marketing und App
- Einfacher: Ein Deployment, ein SSL-Cert
- Links von App zurück zu Blog/Docs stärken die Haupt-Domain

**Für uns:** `ainary.com/analyze` oder `ainary.com/app` — NICHT `app.ainary.com`

## 2. Psychologie der Post-Login Experience

### Das "Aha-Moment" Prinzip

> "Der Aha-Moment ist der Punkt, an dem ein User zum ersten Mal den Wert des Produkts erkennt."
> — Appcues Research

**Entscheidend:** 3-Step-Onboardings haben 72% Completion Rate. 7-Step-Onboardings: nur 16%.

→ **Weniger Steps = mehr Aktivierung.**

### Kahneman's Zwei Systeme

- **System 1 (schnell, emotional):** Der User soll SOFORT fühlen "das ist für mich". Design, Klarheit, ein einziger Fokus.
- **System 2 (langsam, analytisch):** Kommt erst wenn der User den Report liest. Nicht beim Login.

→ **Post-Login muss System 1 ansprechen.** Ein Input-Feld. Eine Frage. Kein Nachdenken nötig.

### Endowment Effect

Sobald ein User seinen ERSTEN Report hat, fühlt er sich als "Besitzer". Der Report ist SEINER. Das macht Churn schwerer.

→ **Ziel: User soll in < 5 Minuten nach Login seinen ersten Report haben.**

### Zeigarnik Effect

Menschen erinnern sich an unvollständige Aufgaben besser als an abgeschlossene. Die Loading-Animation ("Agent 3/5 arbeitet...") hält den User gefangen.

→ **Die Wartezeit ist kein Bug, sie ist ein Feature.** Der User kann nicht weggehen während die Agents arbeiten.

## 4. SEO-Strategie: Links zurück zur Homepage

### Warum das klug ist

Jede interne Verlinkung stärkt die Domain Authority. Wenn eingeloggte User von `/app` auf `/blog/how-agents-work` klicken, zählt das für Google.

### Was die Besten verlinken (aus der App heraus):

| Von (App) | Nach (Marketing) | Warum |
|-----------|-------------------|-------|
| Report Footer | "Powered by Ainary" → Landing | Brand Awareness + Backlink |
| Report Footer | "How this was built" → /quality | Vertrauen + SEO |
| Help/Docs Links | /blog/[relevant-article] | Content Discovery |
| "Upgrade" Button | /pricing | Conversion |
| Logo | / (Landing) | Standard, erwartet |
| Daily Brief in App | /daily-brief (öffentlich) | SEO-Seite wird gestärkt |
| Empty State | "Learn how X-Ray works" → /products | Education + SEO |

### Für Ainary konkret:

**In jedem generierten Report:**
- Footer: "Generated by ●Ainary — [How we build reports →](/quality)"
- Shareable: "Share this report" → öffentliche URL mit Login-Wall für Details
- Related: "Read more about [Industry]" → Blog-Artikel (wenn vorhanden)

**Im Post-Login Screen:**
- "Today's Intelligence Brief" → /daily-brief
- "How our agents work" → /quality (bei erstem Login)

**Das ist der Flywheel-Kick:** App → Content → SEO → New Users → App

## 8. Kern-UX-Prinzipien (confirmed by Florian)

### P1: "Jede Sekunde Wartezeit = eine Sekunde Wert-Delivery"
Loading Screen zeigt personalisierte Intelligence basierend auf User-Profil (Industry, vergangene Analysen, Custom Sources). Die Wartezeit VERKAUFT das nächste Produkt.

### P2: "Custom Sources = Can't-Live-Without Feature"
Free: Unsere 5 Plattformen. Pro: Eigene RSS Feeds + Quellen hinzufügen. One-Click Bundles pro Industry ("VC AI Bundle: 10 Feeds"). Switching Cost = kuratierte Quellensammlung. Netzwerkeffekt: Crowdsourced Quality Signals.

### P3: "Wartezeit-Upsell"
"You might also want to analyze: [Volkswagen] [Stellantis] [BYD]" — kontextuell, nicht random. Nächster Credit wird verbraucht.

### P4: "Login = Personalisierung"
Nach Login wissen wir: Industry Focus, vergangene Analysen, Custom Sources. Alles wird personalisiert — Daily Brief, Loading Insights, Empfehlungen.
